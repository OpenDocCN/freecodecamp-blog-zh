# 什么是谷歌文档季？

“文档季”是由谷歌组织的年度项目。它的目标是将技术作者和开源组织聚集在一起，促进开源空间中文档的协作和改进。

这一举措极其重要，因为开源项目的文档为用户提供了一个途径，不仅可以理解项目，还可以为项目做出贡献。

在该计划期间，公认的技术作者花费 3-5 个月的时间来建立新的文档集，改进现有文档的结构，开发急需的教程，或者改进开源组织的贡献流程和指南。

# 谁有资格参与该计划？

根据[谷歌的规则](https://developers.google.com/season-of-docs/docs)，注册时必须年满 18 周岁。你还必须有技术写作的经验，并有资格在你居住的国家工作。

如果你符合这些条件，那么你可以申请参加谷歌文档季计划。

# 参与的好处

参与“文件季”计划有很多好处。有些是:

### 你可以得到津贴

在你完成谷歌文档季计划后，你会从谷歌获得一笔津贴。重要的是要注意，这种津贴是可选的，所以你必须表明你想在申请阶段得到报酬。

津贴金额是根据您的所在地计算的。参见[此处](https://developers.google.com/season-of-docs/docs/tech-writer-stipends)了解关于津贴的全部细节。

### 你提高了你的技能

参加谷歌文档季计划的另一个明显的好处是，它可以让你提高作为技术写作者的技能。

### 你扩大了你的关系网

除了作为一名技术作家得到提高之外，你还有机会与在开源领域做出惊人成就的人一起工作。

就我而言，我得到了一个与 VideoLAN 组织合作的机会，这意味着我必须与该组织的主席和项目的主要贡献者密切合作。

### 你可以得到推荐和推荐

申请任何工作时，有一封好的推荐信往往有助于强化你的申请。

在 Docs 项目季中与导师密切合作提供了一个获得大量推荐和推荐的机会。

### 你获得了未来的机会

如果你准备申请谷歌的技术写作职位，在简历中加入你的文档经验会让你比其他申请人更有优势。

### 你成为一个开源贡献者

除了上述所有的好处，参加 Docs 季计划意味着你可以为开源项目做贡献。这有助于世界揭穿开源贡献只对软件开发者有意义的刻板印象。

# 提升 GSoD 申请的 5 个技巧

这里有 5 个技巧可以帮助你准备和创建一个强有力的申请。

### 在您的博客上创建和发布文章

在谷歌文档季项目中，被接受为技术写作者的最重要的标准之一是能够证明你有写作经验。

谷歌和你想从事的项目的拟议导师需要确信你是从事该项目的合适人选。这意味着他们希望看到大量你写得很好的文档和文章。

值得庆幸的是，现在创建一个博客或者在你已经存在的博客上发布更多的内容还为时不晚。

如果你已经有了一个博客，在你提交提案之前、期间和之后发表更多写得好的文章是一个好主意。如果你没有任何博客，现在使用 [Hashnode 的 Devblog](https://hashnode.com/devblog) 或你喜欢的平台创建一个并发表写得很好的文章还为时不晚。

### 选择正确的开源项目

老实说，我不能过分强调选择正确的开源项目的重要性。这是因为它对你进入这个项目有很大的作用。

以下是我对选择合适项目的想法:

*   浏览所有[批准的项目](https://developers.google.com/season-of-docs/docs/participants)，尝试选择至少 5 个你感兴趣的项目。
*   选择至少 5 个项目后，再次浏览，将项目减少到最多 3 个项目，最少 2 个项目。
*   虽然在申请阶段专注于一个项目可能是合理的，但我鼓励你申请至少两个项目，以增加至少一个项目被接受的机会。
*   对于你选择的每个项目，加入组织的沟通渠道，让项目的维护者知道你对那个项目的兴趣。这样，你就能在自己和导师之间建立一种关系。此外，请注意，导师不会因为人们拥有必要的技能就选择他们。他们选择申请人也是因为他们相信自己也能与之共事。因此，当你给导师发信息时，要明确你到底需要什么，最重要的是，不要因为他们没有立即回复而感到沮丧，因为大多数导师都是志愿者，他们有其他全职工作。
*   不要等着别人用勺子喂你。对这个项目进行大量的研究。发现并理解你需要知道的一切，如果你有任何问题，向该项目的建议导师寻求帮助，或者询问该社区的其他成员。
*   最重要的是，写下你在研究过程中发现的任何有助于你被项目录用的东西，或者有助于改进你的提案的信息。

### 写一份建议书

很多人不知道这一点，但写提案是至关重要的。它让你清楚地了解项目的目标，完成项目的时间表，你需要知道什么，等等。

除此之外，写建议书还可以向你的导师展示你的项目目标，为什么你认为你是这个角色的合适人选，你的经验，项目时间表等等。

写完你的提案后，把它送给项目的导师审阅。他们的评审将帮助你从导师的角度了解项目需求。我在参考资料部分添加了一个链接，链接到我的 Google 文档季提议。

### 至少对开源组织做出一点贡献

“文档季”计划不像谷歌“代码之夏”那样，要求你至少为感兴趣的开源项目做出一份贡献。但是做出贡献肯定能提高你被录取的几率，所以努力去做吧。

### 在申请截止日期后继续活跃在社区中

提交申请后，试着在开源组织中保持活跃。这向导师们展示了你对这个项目和整个开源社区的热情。

# 我的 GSoD 经历

在去年的 GSoD 项目中，我有机会与 VideoLAN 组织合作更新(重写)VLC 用户文档的项目。

在我的提议被 VideoLAN 组织接受后，我和我的导师们就我在谷歌文档季计划期间应该实现的目标达成了一致。

这些是我的目标:

1.  重组文档。
2.  更新文档以适应 VLC 的现代版本。
3.  使用 Sphinx 和 ReadtheDocs 将用户文档迁移到 Gitlab。
4.  删除过时的图像和信息。
5.  重写用户文档，使其易于理解。
6.  使用 Sphinx 国际化设置它进行翻译。

我对 VideoLAN 组织和 VLC 有了更多的了解，我发现 VLC 有更多我不知道的功能，我学会了如何使用 Sphinx 文档平台和重组文本。

我的技术写作技能也有了很大的提高。总的来说，参加 Google Season of Docs 计划是我在 2019 年遇到的最好的事情之一。
