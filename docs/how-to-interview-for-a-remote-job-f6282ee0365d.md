# 远程工作面试时要问的 6 件事

你们中的许多人可能已经注意到，最近有一个新的“额外津贴”或“类别”在工作列表中出现得更多了。

![1*Wep0oefiv7RnNTeGSRpFoQ](img/ac05d119b74a5d4eb217c3eb5c08c3b4.png)![1*OZsyerDvykQpZ8FnonyUlg](img/cd1acf2cf773b175a5ade3a8756876f5.png)![1*_-1_rh0Ib0rAl8CX5XPrTw](img/ee7626ac834d31d40afd75433f830e84.png)

你可能还会注意到，这些工作看起来真的很有趣，可能比你可接受的通勤半径内现有的工作更有趣。我认为每个人都应该有机会在自己热爱的地方工作。如果我开了一家公司，我的第一个标准是找到对我们所做的事情有热情的人，其他所有标准都遵循这个标准，而不是领导这个标准。

那么对于这些梦寐以求的远程职位，该如何面试呢？让我们来看看你可以向潜在的远程雇主提出的一些指导原则和问题。

#### 完全远程位置

我工作过的大多数公司都以某种方式支持远程工作，有些公司比其他公司更好，但是对于雇主和远程员工来说，显然有一个学习曲线。

通常，列为远程的作业是完全远程的。这意味着你可能会飞去见每个人一周，但之后你可能只能每季度见他们一次，甚至更少(显然除了视频聊天)。有时候你甚至不需要和每个人见面。沟通是通过 Slack/Hipchat、视频聊天(如 Skype 或 Google Hangouts)、项目管理工具以及电子邮件和电话来完成的。

#### 要找什么

我相信还有很多更好的东西可以寻找，我希望你能评论并添加更多到这个列表中。

1.  "您目前有多少全职远程人员为您工作？"
    好:“我们大约有 25%的员工远程办公。”
    不好:“你会是第一个，真的，我们只是列出了远程以获得更多的申请人。你对搬迁持开放态度吗？”
2.  “您通常如何与您的远程团队沟通？”
    好:“我们大部分时间都在聊天，但我们通常会在早上进行视频聊天，除了 Fred，他的游艇上没有太多带宽”
    坏:“我们使用 XYZ 项目管理软件和电子邮件，仅此而已。”
3.  “现场团队每天使用 Slack/Hipchat 的时间有多长？”好:“一整天，他们坐在一起做那件事，但是他们在聊天室聊天。”
    不好:“现场团队实际上不需要使用聊天工具，因为他们都在现场”
4.  "你是如何与远程团队成员一起解决“协作问题”的？"
    好:“如果有绘画，我们有一个应用程序，任何人都可以在上面画画，任何人都可以看到画的是什么。如果需要的话，我们还会让所有人进行视频通话，定期讨论解决问题。”
    不好:“我们通常不会让每个人都打电话。现场人员在白板前工作，然后开始实施”
5.  "人们通常什么时候结束那天的工作？"
    好:“通常在 5-6 点左右，取决于他们开始的时间，事情通常在下午 6:30 就结束了，但偶尔会有 GIF 或文章发布”
    坏:“聊天在与团队沟通方面非常有效，我们可以全天候工作”
6.  “您与远程员工一起使用多少工具？”好:“我们有针对特定意图的特定工具。我们使用 github 进行项目管理，使用 slack 进行异步通信，使用 Skype 进行视频通话"
    不好:“我们使用电子邮件、一些项目管理工具、bitbucket 和 github 以及一些内部 gitlab repos，一些人在 AIM 上，一些人在 IRC 上，其他人使用 Slack，有一个小团队专门使用电子邮件……”

#### 结论

主要想法是看看管理层是否真的认真利用远程工作给他们带来的好处。如果管理层不 100%支持远程员工，那么你可能有一份梦想中的工作，从事梦想中的产品，但当你试图在公司内部导航时，事情可能会变糟。

你应该试着采访几家支持远程员工的公司，看看他们如何以不同的方式回答你的问题。您将能够分辨出那些在其文化中真正支持远程的公司与其他正在转变或没有真正支持这一概念的公司之间的区别。

这篇文章实际上是为了帮助正在面试远程职位的人，请评论你的想法和暗示，你认为这样可以避免某人进入一个粗糙的远程职位，而不是一个员工可以成长和发展的远程职位！
