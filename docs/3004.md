# 如何从初级开发人员成为中级开发人员

> 原文：<https://www.freecodecamp.org/news/how-to-go-from-junior-developer-to-mid-level-developer/>

获得第一份开发人员的工作是许多新程序员的目标。但是我发现很少有开发人员想在他们的职业生涯中迈出下一步。

我发现自己处于这个位置，所以我制定了一个计划来推进我的职业生涯。五个多月来，我学会了获得三份工作机会所需的一切，这份工作机会几乎让我的薪水翻了一倍。这不容易，但绝对值得。

以下是我的做法，以及你可以采取的步骤。

但首先，由于这是一篇深入的文章，这里有一个我的建议的快速总结。

## 7 步初级开发人员职业指南

1.  决定你的目标。你想要什么样的工作，什么时候想要？如果你努力的话，半年的业余学习可以学到很多东西。
2.  为你最终想要得到的工作找到 10-20 份当前的工作描述。创建一个所有需求的列表，列出每个需求出现的次数。
3.  找到这些工作通常需要的 3-5 项主要技能，而这些技能是你目前没有的。
4.  利用课程来提高你学习新技能的速度。课程可以是一个很好的主意，因为它们可以节省你的时间和精力。
5.  在你学会所有东西之前申请工作。面试是一项你也应该练习的技能。
6.  用你所学的知识来构建项目——这是确保你巩固技能的最好方法。
7.  得到你努力得到的工作。

如果你想成为一名全栈开发人员，并且想知道你需要学习什么，我有一本免费的指南，介绍你需要的技能和工具。

## 目标

我计划在东南亚旅行 3-6 个月，我知道我想在回国后找到一份更好的工作。我的初级 JavaScript 工作让我明白了三件事:

*   我喜欢编程，不仅是作为一种爱好，也是一份工作。
*   我喜欢后端开发工作。它更符合逻辑，这是我非常喜欢的。
*   我不太喜欢棱角分明。

有了这些知识，我开始为旅行结束时我想去的地方设定一个目标，以及为了达到那个目标我需要学习些什么。

我的目标是成为一名****-**级别的全栈开发者**** 。**

## **侦察**

**为了成为我想成为的中级全栈开发人员，并找出我需要改进的地方，我寻找我最终真正想得到的工作。**

**首先，我看到了这些工作对候选人的要求，以及他们所说的理想。我列出了所有技能，然后按照它们在工作描述中出现的次数进行排序。**

**有了这个列表，我看到了我还不具备的排名最高的技能。你不需要做清单上的每一件事来得到这些工作。平均而言，一名新员工只具备该职位所需技能的 70-80%。大多数时候，真正的交易破坏者只是几个关键领域。我们的目标是达到那个水平，然后任何其他技能都是一个奖金。**

**通过这项研究，我列出了我正在寻找的大多数工作都需要的东西:**

*   **对 JavaScript 有深刻的理解**
*   **至少一个前端框架(通常是反作用的或有角度的)**
*   **创建 REST API 的能力**
*   **了解一个或多个数据库**

**这个过程真的可以用于任何职业的任何人。但是它在开发中非常有效，因为通常有一个角色“需要”的特定技能列表。**

## **制定计划**

**![_1-2](img/b4990fe42c5d0fa03b7a734fe2ef2756.png)**

**有了这些信息，我知道我需要在几个关键领域提升自己的水平:**

1.  **我需要读完 **你不知道 JS** 。这一系列书籍帮助我真正理解了 JavaScript 的基础知识，所以我认为这对学习更高级的东西很有帮助。**
2.  **我需要变得非常擅长一个前端框架。我研究了 Angular、Vue 和 React，并决定使用 React。我以前做过一点 React 并喜欢它，我不是 Angular 的粉丝，许多招聘广告都没有要求 Vue。**
3.  **虽然我几乎可以从头开始创建一个 express API，但我不是很有信心，只能做基本的工作。我需要用这个全面提升我的游戏，尤其是如果我想有一份优先考虑后端工作的工作。**
4.  **我可以继续使用关系数据库(postgreSQL)或者学习一个新的数据库。我决定学习 MongoDB，因为它非常受欢迎，并在我的简历中添加了 NoSQL 数据库。**

**我不确定我会旅行多久，所以我决定这样计划我的学习:**

1.  **边读 **边学 React(和 Redux)你不知道的 JavaScript** 。**
2.  **更好地进行 API 开发并学习 MongoDB。**
3.  **用我学到的技能做点东西。**
4.  **学习其他的东西。**

**我这样做是有充分理由的。如果我在三个月后结束旅行，完成了第一项，但第二项还在进行中，那么我仍然可以申请中级开发人员的工作——他们只是被限制在主要是前端工作。**

**如果我先做后端学习，3 个月后我仍然不会对任何前端框架很有信心。这可能是每个中层工作的症结所在。我可以尝试申请一个仅后端开发人员的角色，但他们似乎都希望比我有更多的经验。**

# **学习 React 和 Redux，阅读*你不知道的 JS***

**![Image for post](img/cb6164256abb3c3f18f0b9a6fd905edc.png)**

**我以前使用 Youtube 视频和普通网络教程做过一点 React，但这次我想确保我正确地学习了它。这让我从 Udemy 获得了 [**React JS 和 Redux:掌握 Web App**](https://www.udemy.com/react-js-and-redux-mastering-web-apps) 课程。**

**这个课程非常精彩，我在 React 上的一点点经验帮助很大，但是我没有使用 Redux 的经验。在本课程结束时，我觉得我已经可以构建一个相当复杂的网站，并且对 React 的语法和格式以及 Redux 的数据控制都很有信心。**

**这个课程花了我大约两周的时间来完成，但是我每周花一到两个小时来完成它。我在亚洲度假，所以这不是我的首要任务。**

**除了在我有笔记本电脑的时候学习课程，我还阅读了《你不知道的 JS 系列的最后几本书。我把在线版的 Github 储存在手机里，在攀登间隙休息的时候读了一遍。**

**有了一些真实世界的经验，我比上次尝试阅读这些书时明白了很多。像承诺这样的东西，如果你从来没有用过，真的很难理解。**

# **更好地学习 express 和 MongoDB**

**现在我觉得创建前端很舒服，是时候学习我的后端技能了。**

**我再次决定参加一个业余课程。我发现，因为它们是完整的包，所以事情进展得很好，并且倾向于覆盖整个过程。当然，你可以在 Youtube 或其他在线教程上免费学习所有相同的东西，但它们似乎从来没有涵盖一个完整的主题。**

**下一个问题是学哪门课程。有免费的也有付费的，有短的也有长的。**

**我决定投资自己，买一门课程(对于如此大量的信息来说，10 美元是一笔很小的投资)，我设法将范围缩小到大约 3 门课程。它们的长度从 7 小时到 25.5 小时不等。**

**我被 7 小时的课程所吸引——我可以在一两周内完成，同时还能享受我的假期。25 小时的课程显然需要更长的时间，但似乎涵盖了更多的主题。在旅行的这一点上，我决定只要我的银行余额允许，我就会旅行，所以我选择了 25 小时的课程。**

# **申请工作**

**我参观的下一个地方是老挝的一个偏远的登山区。这时，我知道我在老挝呆了两周，然后在回家前又在中国呆了大约四周。该去申请工作了。**

**如果你不像我一样有一个严格的截止日期，那么我会试着设定一个。这真的有助于激励你完成你的项目或开始找工作(没有人喜欢搜索工作公告板)。**

**如果你不想设定一个截止日期，那就在你开始申请工作的时候设定一个学习点。**

**这不应该在你学完所有东西之后！学习面试也是一种技能，你应该尽可能经常地获得这方面的经验。开始申请你不太热衷的工作来练习整个过程。**

## **进步**

**在接下来的一个半月里，我申请了 50 多份工作，通过电子邮件和 LinkedIn 与大约 25 人进行了交谈，并设法在我回来后的一周内获得了五次面试机会。**

**我的大部分互动都是通过电子邮件进行的，但是如果有人想直接打电话，我会尽最大努力满足他(我提前了八个小时)。**

**我认为进行一次真正的对话是改善你和他人之间关系的好方法，增加了他们邀请你面试或推荐你工作的机会。**

**如果你觉得和别人打电话或用 Skype 聊天不舒服，那么你可以坚持用电子邮件，但我认为你错过了提高机会。**

**这也给了我一个机会，把我在过去 4 个月里学到的东西付诸实践，并建立一些展示我能做什么的东西。**

# **面谈**

**既然我已经安排好了这五次面试，我必须把它们转换成工作邀请。**

## **准备**

**我写过关于[准备 JavaScript 面试](https://www.freecodecamp.org/news/5-top-sites-for-javascript-interview-preparation-71b48e9a6c8a/)和[掌握面试流程](https://medium.com/@samwsoftware/how-to-secure-the-job-of-your-dreams-by-smashing-your-interview-61f38b7cdd0e)的文章。我确保我勾选了每一个方框。**

**我知道我想要强调的所有要点，如何回答关于我的弱点的问题，以及每家公司都做了什么。**

## **面试**

**尽管我天生内向，但我一直很擅长面试。我不得不教自己表现得更加外向，与面试我的人交往。**

**如果你发现自己在面试中表现得害羞和拘谨，我建议你像练习其他技能一样练习这一点。找到一个好的资源(charisma on command 是一个很棒的 Youtube 频道)，然后重复你所学到的。从朋友和家人给你假面试开始，然后试着去找那些你不太熟悉的人，他们可能更像真正的开发人员。**

**我在四分之三的面试中进行了技术测试(另外两次是在网上进行的)，我的准备得到了回报。有一些小失误，但是我通过我的思考，很好地完成了所有的任务。**

**你的面试也是一个机会，可以更多地了解这家公司，以及在那里工作会是什么样子。一家公司可能在纸面上看起来很棒，但并不像宣传的那样，或者可能正好相反。选择合适的公司是一个重大决定，所以确保你询问了所有你需要的信息。**

# **摘要**

**我申请了很多工作，得到了五次面试机会。为这些面试做好准备后，我得到了 4 份工作机会，还有 2 份我想从中做出选择。我和两家我最喜欢的公司谈好了工作，最终在一家大公司得到了一份很棒的工作。**
