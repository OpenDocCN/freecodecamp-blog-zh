# 错误 403 禁止解释-我该如何修复这个 HTTP 错误代码？

> 原文：<https://www.freecodecamp.org/news/error-403-forbidden-explained-how-can-i-fix-this-http-error-code/>

所以你遇到了一个 **403 禁止**错误，你想知道这是什么意思。

此错误是一个 HTTP 状态代码，意味着您被禁止访问您试图访问的页面或资源。除非你是创建网站的人，否则往往无能为力。然而，有一些事情可能会有所帮助。

有可能是网站的创建者正确设置了权限，故意禁止您访问该页面。但是该错误也可能表明网站设置不正确。

这里有一些事情你可以尝试修复 403 禁止错误。

### ？验证并刷新

首先验证 URL 是否正确，然后刷新页面。当你在网站上遇到任何错误时，这是你应该做的第一件事。

如果有人试图访问服务器上的目录而不是文件(如 HTML 文件)，大多数 web 服务器都会返回 403 禁止错误。因此，您可能输入了错误的 URL，并试图访问某个目录。

### ？清除浏览器缓存

该页面的缓存版本可能会导致该问题。以下是在大多数浏览器上清除浏览器缓存的快捷键:

*   视窗:`CTRL + F5`
*   苹果/苹果:`Apple + Shift + R or Command + Shift + R`

### ⌨️登录

您尝试访问的页面可能需要您登录。如果是这样，请确保登录以获得额外的访问权限。

### ？清除 cookies

清除浏览器的 cookies 有时会有所帮助。如果站点通常需要登录，并且注销和登录都不能解决问题，这一点尤其正确。

### ？联系网站

可能是网站设置不正确，网站的创建者不知道。其他人可能会遇到同样的错误。试着找到网站的联系方式，让他们知道这个问题。这可能是他们的一个简单的解决办法。

### ？️回来以后

通常 403 禁止错误是由网站的问题引起的。有可能是网站开发者目前正在解决这个问题。如果您稍后再试，问题可能会得到解决。

## 仅适用于 web 开发人员

如果您是给出 403 禁止错误的页面的创建者，那么您的工作就是修复服务器上的错误。该错误最常见的两个原因是没有索引页和权限不正确。

确保在显示错误的位置有一个名为 index.htm 或 index.php 的文件。例如，如果 URL `https://www.freecodecamp.org/forbidden`显示错误，确保服务器上名为`forbidden`的目录中有一个 index.htm 或 index.php 文件。

接下来要研究的是对产生错误的文件的权限。权限通常应该是这样的:

*   文件夹:755
*   静态内容:644
*   动态内容:700
